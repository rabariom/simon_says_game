# simon_says_game
i made simon says game using html, css, and javascript to polish my knowledge, logic building and skills of these technologies.

A simple, interactive web-based Simon Says Game built using HTML, CSS, and vanilla JavaScript. The game tracks the player's current score and preserves their highest score across sessions using browser local storage.

# Features

*Keyboard Initiation: The game starts or restarts when any keyboard key is pressed.
*Visual Sequences: Randomly generates a sequence using four distinct colored buttons (Green, Blue, Red, Yellow).
Distinct Flash Effects: 
    * Game-generated sequences trigger a **white** flash (`.flash`).
    * Player clicks trigger a **black** flash (`.userflash`).
*Score Tracking: Displays the current level during gameplay.
*Persistence: Tracks and stores the highest score using `localStorage`.
*Game Over Warning: Flashes the entire screen background **red** for 250ms when a sequence is failed.

# Project Structure

The repository consists of the following files:
* `index.html` - Links the stylesheet and script, and defines the structural layout of the game UI.
* `style.css` - Manages the visual design, button layouts, and color flash effects.
* `app.js` - Handles the game logic, sequence generation, event listeners, state tracking, and high score management.

# Technical Details

# State Variables (`app.js`)

| Variable | Type | Description

`gameSeq` | Array | Stores the randomized pattern sequence generated by the game. 
`userSeq` | Array | Stores the pattern sequence entered by the player during the level. 
`started` | Boolean | Tracks whether the game session is active or in a reset state. 
`level` | Number | Tracks the current active game level (score). 
`highestScore` | Number / String | Retrieves and tracks the top score from browser `localStorage`. 

#Styling & Layout (`style.css`)

* Buttons are rendered with fixed dimensions (`200px` height and width), a `10px solid black` border, and a `20px` border radius.
* The buttons are grouped inside a flexbox container (`.btn-container`) centered on the page, aligned in two structural rows/lines:
    * **Line 1**: Green, Red
    * **Line 2**: Blue, Yellow

# Game Logic & Rules

1. Start: Pressing any key transitions the game state from `started = false` to `true` and initiates the first level (`levelUp`).
2. Gameplay: 
    * A color is randomly chosen from an array of four colors (`["green", "blue", "red", "yellow"]`) using `Math.floor(Math.random() 4).
    * The selected button flashes white, and the color is appended to `gameSeq`.
    * The player must click the buttons in the exact sequence generated so far.
3. Validation:
    * Every button press is verified at its current index against `gameSeq`.
    * If the user correctly completes the sequence (`userSeq.length == gameSeq.length`), the next level is triggered after a 1000ms delay.
4. Game Over:
    * If a wrong button is pressed, the current level is compared with `highestScore`. If the current level is greater, `highestScore` updates in `localStorage`.
    * The background flashes red, the heading displays the current score and highest score, and all states are reset (`started = false`, sequence arrays cleared, level set to 0).
